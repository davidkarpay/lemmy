# Apollo Eagle Lunar Lander Sim — v1.1 Spec

## Scope rule
Version one point one fixes what currently breaks authenticity. No new content layers, no terrain expansion, and no Apollo Guidance Computer keypunch simulation. If a change is not on this list, it waits for version two.

## Build order
1. Mouse-look fix
2. Cockpit hot zones and switch-state source of truth
3. Remove fake drag and retune realistic inertia
4. Sharpen P63, P64, and P66 phase differentiation
5. Post-run debrief panel

## 1. Mouse-look — make the promise real
- Wire held mouse-look in all cockpit views.
- Pointer lock on press, release on let-go.
- Sensitivity tuned so a full window sweep covers the realistic Lunar Module commander sightline arc.
- Keyboard modifier look stays as the secondary path.
- Constrain to Lunar Module window and panel sightlines in cockpit views.
- Third-person view stays unconstrained.

### Acceptance
Dragging the mouse in Commander Window view actually moves the view, and the view stops at the real Lunar Module window frame.

## 2. Physical cockpit hot zones for the critical switches
Promote these checklist items from flag flips to clickable, stateful cockpit controls with visible position and guarded covers where authentic:
- Descent engine arm
- Landing radar power and antenna mode
- Guidance control: Primary Guidance and Navigation System or Abort Guidance System
- Attitude control mode per axis: automatic, attitude hold, off
- Program selector for P63, P64, P66
- Undock and separation arm plus execute
- Engine stop after contact light

### Source of truth rule
Each switch position is the source of truth. Checklist completion reads from switch state, not the other way around.

### Shortcut rule
Shift plus number shortcuts continue to work and visually throw the corresponding switch.

## 3. Remove the fake drag term and retune realistic inertia
Delete the velocity-damping placeholder in the descent loop.

### Realistic mode requirements
- Translational Reaction Control System control is discrete: jet on or off, fixed thrust, fixed minimum impulse.
- Reaction Control System and descent throttle have a small response lag.
- Attitude hold is not magical: it nulls rates within a deadband but does not cancel commanded rates or correct drift for free.
- Inertia is preserved aggressively.
- Overshoot is the default outcome of overcontrol.

### Arcade rule
Arcade keeps its current softening. The new realism lives entirely on the Realistic side.

## 4. Sharpen P63, P64, P66 so they feel operationally distinct
### P63 braking
- Heads-down feel
- Guidance owns throttle and attitude target
- Pilot monitors
- Display emphasizes downrange, altitude solution, and engine performance
- Minimal pilot authority beyond monitoring and abort arming

### P64 approach
- View shifts to look-angle reference
- Landing-point designator becomes the primary tool
- Pilot can bias the target laterally and downrange in discrete clicks
- Computer still flies, but the player owns where it is flying to

### P66 manual hover
- Pilot owns attitude fully
- Throttle is a rate-of-descent command, not direct throttle
- Hover bleeds fuel visibly fast
- No auto-leveling assist in Realistic mode
- Cueing changes from solution-driven to pilot-driven

### Acceptance
A player who knows the phases can tell which one is active from the cueing and workload alone, with labels hidden.

## 5. Post-run debrief panel
On landing or failure, show a single screen that reports:
- Touchdown verdict and which limit decided it: tilt, vertical rate, lateral rate, target offset, fuel out, or mission-ending fault
- Fuel budget breakdown: nominal versus actual, with the largest waste event flagged
- First overcontrol moment timestamp and axis
- Checklist misses that actually propagated into degraded state, with the downstream effect named
- Phase grades for P63, P64, and P66 separately

### Presentation
- Authentic mode uses mission-report style
- Training mode adds plain-English explanation per row
